# 給与自動計算 — 配布・運用マニュアル

個人利用のローカルアプリを、Python 不要で第三者へ配布できる Windows アプリ
（`setup.exe`／ユーザー単位インストール／起動時の自動更新付き）にするための手順書です。

---

## 0. 全体像

```
[開発PC] payroll_app.py 他  ──build──▶  dist\給与自動計算.exe
                                   │
                                   ├─ Inno Setup ─▶  給与自動計算_setup.exe（配布）
                                   │
                                   └─ zip 化 ─▶  給与自動計算.zip（自動更新の配信物）

[配信] GitHub Releases に  給与自動計算.zip  と  version.json  を公開
[利用者] setup.exe で導入 → 起動時に version.json を確認 → 新版があればワンクリック更新
```

- **データ保存先**: `%APPDATA%\給与自動計算\payroll.db`
  （インストール先ではないため、更新・再インストールでも消えません。既存DBは初回自動移行）

---

## 1. ファイル構成

### 既存（変更あり）
| ファイル | 変更内容 |
|---|---|
| `payroll_app.py` | バージョン表示・起動時の更新確認・「更新を確認」ボタンを追加 |
| `payroll_db.py` | DB を `%APPDATA%\給与自動計算\` に保存（既存DBは自動移行） |
| `payroll_calc.py` / `payroll_pdf.py` | 変更なし（既存ロジック維持） |

### 新規
| ファイル | 役割 |
|---|---|
| `version.py` | バージョン番号・version.json の URL |
| `app_paths.py` | 実行パス／データ保存先の解決（frozen 対応） |
| `updater.py` | 更新確認・ダウンロード・上書き・再起動 |
| `payroll.spec` | PyInstaller ビルド定義（onefile/GUI） |
| `installer.iss` | Inno Setup インストーラ定義（ユーザー単位） |
| `version.json` | 配信用テンプレート |
| `requirements.txt` | 依存ライブラリ |
| `build.bat` | ビルド補助スクリプト |

---

## 2. ビルド手順（exe 化）

### 前提
- Python 3.10+（開発PCのみ。利用者には不要）

### 手順
```
1. このフォルダで build.bat をダブルクリック
   （または）コマンドプロンプトで:
       python -m pip install -r requirements.txt pyinstaller
       pyinstaller --noconfirm payroll.spec

2. 生成物: dist\給与自動計算.exe   ← Python 不要で動く単一exe
```

> 補足: reportlab の日本語フォントデータ・openpyxl は `payroll.spec` の
> `collect_all` で同梱済みです。

---

## 3. インストーラ作成（setup.exe）

### 前提
- [Inno Setup](https://jrsoftware.org/isdl.php)（Unicode 版）をインストール

### 手順
```
1. Inno Setup Compiler を起動
2. installer.iss を開く
3. メニュー [Build] → [Compile]
4. 生成物: Output\給与自動計算_setup.exe   ← これを配布
```

### インストーラの特徴
- **管理者権限不要**（`PrivilegesRequired=lowest`）
- インストール先: `%LOCALAPPDATA%\Programs\給与自動計算`（**ユーザー単位**・書込可能）
  → 書込可能なため自動更新が機能する
- スタートメニュー／デスクトップ（任意）にショートカット作成
- アンインストールしてもユーザーデータ（`%APPDATA%\給与自動計算`）は残す

## 4. GitHub Releases を使った更新配信手順

### 4-1. 初回準備
1. GitHub にリポジトリを作成（private 可。ただし下記URLに匿名アクセスできること）。
2. `version.py` の `UPDATE_URL` を、公開する version.json の RAW URL に書き換える。
   ```python
   UPDATE_URL = "https://raw.githubusercontent.com/<owner>/<repo>/main/version.json"
   ```
   ※ private リポジトリは RAW に認証が必要なため、**public リポジトリ**または
     別の常時公開URL（自社サーバ等）を推奨。

### 4-2. リリースのたびに行う作業
バージョンを `1.0.0 → 1.1.0` に上げる例:

1. **コードを更新**し、`version.py` の `__version__` を `"1.1.0"` に変更。
2. **ビルド**（第2章）→ `dist\給与自動計算.exe` を生成。
3. **zip 化**: `給与自動計算.exe` を `給与自動計算.zip` に圧縮。
   - zip の中身は `給与自動計算.exe`（単一ファイル）。フォルダで包んでもOK
     （updater が自動で中身を判定）。
4. **インストーラも更新**（新規利用者向け）: `installer.iss` の `AppVersion` を
   `1.1.0` にして再コンパイル → `給与自動計算_setup.exe`。
5. **GitHub Release を作成**:
   - タグ: `v1.1.0`
   - 添付（Assets）: `給与自動計算.zip` と `給与自動計算_setup.exe`
6. **version.json を更新**してリポジトリへ push（`main` 直下）:
   ```json
   {
     "version": "1.1.0",
     "download_url": "https://github.com/<owner>/<repo>/releases/download/v1.1.0/給与自動計算.zip"
   }
   ```

> `download_url` は Release のアセット URL。zip を新タグに添付するたびに URL が変わるので、
> version.json の `download_url` も合わせて更新する。

### 4-3. 利用者側の挙動（自動）
1. アプリ起動 → 約1秒後に version.json を取得。
2. `version.json.version` > 現在版 なら以下を表示:
   ```
   新しいバージョンがあります
   現在：1.0.0
   最新：1.1.0
   今すぐ更新しますか？
   ```
3. 「はい」→ zip をダウンロード → アプリ終了 → 上書き → 自動再起動。
4. ネット不通・取得失敗時は**何も表示せず**通常起動（業務を妨げない）。
   手動確認は「その他項目設定」タブの **［更新を確認］** から可能。

---

## 5. 更新の仕組み（技術メモ）

- `updater.apply_update()` が zip を `%TEMP%` に展開し、PowerShell ヘルパー
  （`apply_update.ps1`）を**分離プロセス**で起動。
- ヘルパーは「本体プロセスの終了を待つ → インストール先へ上書きコピー → 再起動」を実施。
  日本語パスに対応するため UTF-8(BOM) の PowerShell で処理。
- 自動更新は**配布版（frozen）でのみ**動作。開発時のスクリプト実行では無効。

---

## 6. 運用上の注意

| 項目 | 内容 |
|---|---|
| データ場所 | `%APPDATA%\給与自動計算\payroll.db`。バックアップはこのファイルをコピー |
| 既存データ移行 | 旧版（exe同階層DB）から初回起動時に自動コピー。元DBはそのまま残る |
| バージョン番号 | 必ず単調増加（例 1.0.0 → 1.0.1 → 1.1.0）。数値比較で判定 |
| 自動更新が不要な配布 | `version.py` の `UPDATE_URL` を空運用（version.json を公開しない）にすれば通知は出ない |
| 署名なしの警告 | コード署名証明書が無いと SmartScreen 警告が出る場合あり。「詳細情報→実行」で回避可。気になる場合はコード署名を検討 |

## 7. クイックリファレンス

```bat
:: ビルド（exe）
build.bat

:: PyInstaller を直接叩く場合
pyinstaller --noconfirm payroll.spec

:: 成果物
dist\給与自動計算.exe                 ← 単一実行ファイル
Output\給与自動計算_setup.exe         ← 配布用インストーラ（Inno Setup）
給与自動計算.zip                      ← 自動更新の配信物（exe を zip 化）
```
